SQL Injection (SQLi) Prevention

Concept: SQLi occurs when user input is incorrectly passed to a database, allowing attackers to manipulate queries.

Defense: Always use Parameterized Queries (Prepared Statements). Never concatenate user input directly into SQL strings.

Brute Force & Rate Limiting

Concept: Tools like Hydra rapidly guess passwords. They rely on the server allowing unlimited attempts.

Defense: Implement account lockouts, IP-based rate limiting (e.g., max 5 attempts per minute), and enforce strong password policies.
